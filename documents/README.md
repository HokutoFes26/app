# ドキュメント

本アプリは不特定多数のユーザーが利用し、かつ Supabase Pro などの課金サービスを利用しているため、技術的な理解だけでなく運用上の注意点が重要です。

## ドキュメント構成

1.  **[インフラとデプロイ](./INFRASTRUCTURE.md)**
    *   Supabase Pro の管理、GitHub Pages へのデプロイ（`/app` パス）、静的書き出し設定について。
2.  **[ロールとセキュリティ](./ROLES_AND_SECURITY.md)**
    *   User / Admin / StallAdmin の権限設計と、Supabase RLS による保護。
3.  **[運用マニュアル](./OPERATION_GUIDE.md)**
    *   文化祭当日の操作、お知らせ・落とし物・Q&A の更新、トラブル対応。
4.  **[開発ガイド](./DEVELOPMENT.md)**
    *   ローカル開発環境の構築、技術スタック、パス修正スクリプトの仕組み。
5.  **[Supabase セットアップ](./SUPABASE_SETUP.md)**
    *   テーブル定義（SQL）とリアルタイム設定。

## 最優先事項

*   **Supabase の課金:** 本アプリは Supabase Pro プランを前提としています。無料枠に落とすと、同時接続数制限により当日アプリが停止するリスクがあります。
*   **パスの扱い:** GitHub Pages のサブディレクトリ（`/app`）で運用するため、画像やリンクのパスには `getPath` 関数や `basePath` の考慮が必須です。
*   **データのバックアップ:** 投票データやユーザー投稿データは Supabase 上にあります。終了後は速やかにエクスポートすることを推奨します。
